**C言語で`.rt`ファイルのレンダリングを行うレイトレーシングエンジン**

カメラからピクセルごとにレイを生成し、シーン内オブジェクトとの交差判定を行い、Phongライティングモデルで色を決定するレンダリングパイプラインを実装した。球体・平面・円柱のプリミティブに対応し、再帰レイトレーシング（最大深度5）によって反射・屈折を表現。拡張性を意識した設計により、Diffuseに加えてMirror・Metal・Glassの4種類のマテリアルを実装した。

機能：
* レイトレーシングによる3Dシーンの描画（解像度2048×1024）
* プリミティブ：球体、平面、円柱（キャップ付き）
* Phongライティングモデル（環境光・拡散反射・鏡面反射）の実装
* ハードシャドウ
* マテリアル：Diffuse / Mirror / Metal / Glass
* 再帰レイトレーシングによる反射・屈折表現
* カラーライト（光源の色が表面色を変調する乗算混色、複数光源は加法合成）
* `.rt` シーンファイル形式の入力

担当範囲：
* レンダリングのメインループおよびカメラからのレイ生成
* カメラ座標系の構築（視線ベクトルからright/upベクトルの算出）
* Phongライティング（環境光・拡散反射・鏡面反射）の実装
* 球体・平面の交差判定アルゴリズム
* ベクトル演算・RGB演算の基盤実装
* minilibxを用いたウィンドウ処理

## 実行環境

* macOS / Linux
* C言語コンパイラ (clang / gcc)

外部ライブラリへの依存：minilibx のみ（リポジトリに同梱）


## 実行方法

ビルド：
```
$ make
```
実行：
```
$ ./miniRT scene/<scene.rt>
```

`.rt` ファイルにシーン情報（カメラ位置、光源、オブジェクト、マテリアル）を記述して指定する。`scene/` ディレクトリにサンプルシーンを同梱

## 苦労した点

* カメラ座標系の構築

	カメラの視線方向（forward）からright・upベクトルを算出する際、視線方向がワールド上ベクトル（world_up = (0, -1, 0)、画像座標系のためY軸下向きを正に取る）と平行になる場合、外積がゼロベクトルとなりright/upを定義できない問題が発生した（カメラが真上または真下を向くケース）。

	対策として、forwardとworld_upの内積の絶対値が1に近い場合（`fabs(dot(forward, world_up)) > 0.999`）は、world_upを `(1, 0, 0)` にフォールバックする処理を実装した。これにより、任意の視線方向で安定してカメラ座標系を構築できるようになった。
	* `src/utils/calc_screen.c` の `calc_right_vec()`：world_upとforwardの内積判定とフォールバック処理
	* `src/utils/calc_screen.c` の `calc_up_vec()`, `calc_llc()`：right/upからカメラ座標系の構築とビューポートの左下角計算

* 数式の実装と描画結果からのデバッグ

	交差判定やライティング計算で数式を実装する際、計算ミスは「画面が真っ黒」「物体の形状が崩れる」「影の出方が不自然になる」といった描画結果の異常として現れる。値の異常をデバッグで発見しづらく、原因の特定が難しかった。

	対策として、数式をそのまま実装するのではなく、幾何学的な意味に分解して理解した上で実装した。これにより、描画結果の異常から計算式のどの部分に問題があるかを逆算して特定できるようになり、数学を含む処理のデバッグが効率化された。
	* `src/render/intersect_pl.c`, `intersect_sp.c`：平面・球などのオブジェクトとの交差判定ロジック
	* `src/math_original/quadratic_eq.c` の `calc_quad_discriminant()`, `solve_quad_eq()`：二次方程式の判別式計算と解の公式


## 工夫・力をいれた点

* 処理を分離したパイプライン設計による拡張性の確保

	レイトレーシングの実装にあたり、「レイ生成 → 交差判定 → ライティング」の各処理を独立したステージとして分離するパイプライン構造を採用した。これにより、新しいプリミティブやマテリアルの追加時に既存コードへの影響を最小化し、各処理のデバッグを個別に実施可能にした。実際にDiffuseに加えてMirror / Metal / Glassの3種類のマテリアルを後から追加できた。
	* `src/render/render_scene.c`, `ray.c`：ピクセルごとにカメラからレイを生成するメインループ
	* `src/render/hit_scene.c`：オブジェクトリストへのディスパッチによる交差判定ステージ
	* `src/lighting/calc_lighting.c`：マテリアル種別による分岐とライティングステージの統括
